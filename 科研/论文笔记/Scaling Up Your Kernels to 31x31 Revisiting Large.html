# Scaling Up Your Kernels to 31x31/ Revisiting Large Kernel Design in CNNs

作者: Ding, Xiaohan

Ding, Xiaohan, et al. "Scaling up your kernels to 31x31: Revisiting large kernel design in cnns." Proceedings of the IEEE/CVF Conference on Computer Vision and Pattern Recognition. 2022.

# 概述

作者重新研究了大卷积核的设计，提出了使用少量的大卷积核代替多个小卷积核的堆叠，可以使模型具有更好的表现。

文章提出了五条经验性方法，例如使用重参数化（re-parameterized）的大深度卷积(depth-wise convolutions)来设计大核CNNs。

提出了RepLKNet，一种纯粹的CNN网络，它的卷积核大小可以达到31×31

RepLKNet可以使CNNs接近ViTs的表现，在ImageNet和ADE20K数据集上的表现相比于相同模型大小的最新模型非常的有竞争力。

大核CNNs比小核CNNs有更大的有效感受野(effective receptive fields)，更高的形状偏置（而不是纹理偏置）

ViTs中的多头注意力机制(MHSA)层可以获取更大的区域信息，而CNNs却更习惯使用小核，思考能不能使用更大的核来使得CNNs表现接近ViTs。

使用大的深度卷积核(depth-wise convolutions)来代替传统卷积，经过一系列实验，得出了五条经验性的方针：

  1. 在实践中，大卷积核依旧是有效的
  2. identity shortcut 对于大卷积核网络非常的重要
  3. 小卷积核的重参数(re-parameterizing)化有助于解决优化问题
  4. 大卷积核对于下游任务的帮助比ImageNet更大
  5. 大卷积核对于小特征图依旧非常的有效

论文发现大卷积核对下游任务特别有用，比如对于COCO检测数据集和ADE20K分割数据集，模型更小的情况下甚至比Swin Transformers表现更好。

作者认为原因在于大卷积核拥有更大的有效感受野(effective receptive fields)。

# 相关工作

# 大卷积核

自从VGG-Net之后，大核卷积如Inceptions、GCNs变得不再受欢迎，因为大家认为大核卷积在ImageNet的表现不好。

LR-Net提出了空间聚合操作（动态卷积核）来代替传统卷积，但是在卷积核大小达到7×7以后，再增大就会导致表现下降。

Swin Transformers使用了移动窗口注意力，它的窗口大小在7至12之间，可以视为大卷积核。因此有工作使用动态7×7深度卷积核来代替Swin Transformers的多头注意力层。虽然它的思路跟本文很像，但是他没有探究ERFs、大核卷积、表现它们之间的关系，相反，他将ViTs的优越性归结于稀疏连接，共享参数以及动态机制。

还有三个代表性的工作是Global Filter Networks (GFNets)，CKConv和FlexConv。GFNet优化了傅里叶域中的空间连接权值，相当于空间域中的圆形全局卷积。CKConv将卷积核定义为处理顺序数据的连续函数，可以构造任意大的卷积核。FlexConv为不同的层学习不同的卷积核大小，可以像特征映射一样大。虽然他们使用了大核卷积，但是他们没有试图去回答：为什么传统卷积网络比ViTs弱，如何在卷积网络中使用大核卷积。

另外，他们的工作也没有与Swin Transformer进行充分的比较，因此不清楚大核卷积能否像transformers一样具有良好的扩展性。

还有一些同期工作，他们的模型虽然有优越的表现，但是并没有展示更大的卷积核(31×31)的优越性。

# 模型缩放(Model Scaling)

目前模型缩放的重点都还是模型的深度、宽度、输入分辨率、瓶颈比等，而卷积核大小总是被忽略。本文将会说明卷积核大小同样也是一个重要的技巧，特别是对于处理下游任务。

# 如何应用大卷积核

# Guideline 1:DW

可以使用深度卷积核来解决大卷积核带来的计算量大问题。本文提出的RepLKNet的四个阶段卷积核大小从3,3,3,3变为31,29,27,13，参数量只增多了10.4%。

这也带来了一个问题：DW卷积在GPUs上计算不高效，原因在于DW操作的计算和内存访问比例较低。然而作者发现当核变大时，计算密度也会增加。

# Guideline 2: shortcut

identity shortcut对于大卷积核网络非常的有效。作者将MobileNet V2作为基准模型，将它的3×3 layers 改为 13×13，两个模型都训练在ImageNet上，同样的设置，训练100 epochs

# Guideline 3: re-parameterizing

使用并行3×3卷积核来进行结构重参数。

具体方式是，训练时将大核上并行一个3×3卷积核，两个核经过BN后相加。

训练完成后，将小核和BN参数一起合并到大核中，只保留大核。

使用了这样简单的重参数，增大核大小从9到13不再会降低模型表现了（ImageNet核Cityscapes）

# Guideline 4:downstream tasks

大核卷积对于下游任务的表现比在ImageNet分类任务上的表现更出色。

作者认为大的卷积核拥有更大的ERFs，而大的ERFs对于一些下游任务如目标检测、语义分割等非常重要。大的卷积可以获取更多的形状偏置，ImageNet 的图片正确分类同时需要纹理和形状信息，然而人类识别对象更多基于形状信息而不是纹理，因此模型拥有很强的形状偏置对下游任务更加的有效。最近的一些工作也指出ViTs也有很强的形状偏置。

# Guideline 5:small feature maps

就算卷积核比feature maps大，更大的卷积也依旧更有效。

MobileNet V2 最后一阶段的feature map为7×7

# RepLKNet

Stem 为模型第一层，希望通过多个conv layers捕捉更多的细节。

Stages 1-4 包含了多个RepLK块，DW卷积使用一个5×5的卷积核用来重参数，1×1 layers来增加通道深度，提高非线性，每个RepLK Block后面都跟着一个ConvFFN

Transition Blocks 负责增大通道数和下采样

模型可以设置三个超参数，B(RepLK Blocks的数量)，C(通道数)，K(核大小)

# 总结

虽然大卷积核可以显著提高CNNs在ImageNet和下游任务上的表现，但是随着数据和模型规模的进一步扩大，RepLKNets的表现开始落后于Swin Transformers。目前还不清楚这种差距是由次优超参数调优导致的，还是当数据/模型规模扩大时cnn出现的其他一些基本缺陷造成的。作者正在努力解决这个问题。

本文重新讨论了在设计CNN结构时长期被忽视的卷积核大小。作者证明，使用几个大卷积核而不是许多小卷积核可以更有效地获取更大的有效感受野，极大地提高CNN的性能，特别是在下游任务上的性能，并在数据和模型规模扩大时大大缩小CNN和ViTs之间的性能差距。文章的工作能够同时促进CNN和ViTs的研究。一方面，对于CNN，发现表明应该特别关注ERFs，这可能是高性能的关键。另一方面，对于ViT来说，由于大卷积可以替代具有相似行为的多头自关注，这可能有助于理解自关注的内在机制。
