# which

which bash指令

该指令可以查看指令“bash”的绝对路径。

# man

man属于线上查询

查询指令的使用方法与参数说明

# 查看历史记录

cat ~/.bash_history

可以查看以往所使用的命令。

注意，文件中记录的是前一次登陆以前所执行过的指令，而至于这一次登陆所执行的指令都被暂存在内存中，当你成功的登出系统后，该指令记忆才会记录到.bash_history当中！

# 别名设置

# 编写临时别称

输入alias可以直接查看当前系统中有哪些别名，设置别名，可以输入

alias lm='ls-al'

以后输入lm就会执行ls-al了，不过这只是暂时的，退出当前shell 就无法使用了。

# 编写特定用户别称

vim ~/.bashrc
# 将别称写入
# 重新调用
source ~/.bashrc

# 编写系统级别称

与用户级编写过程相同，只是在/etc/profile中写入

vim /etc/profile
source /etc/profile

# type

输入man bash提供的命令，就可以查看命令的使用文档。

使用type 命令就可以查看该命令是内置在bash当中，还是来自外部指令。

# 换行进行编辑

在需要换行的位置打印/，再敲击回车即可

# 组合键

ctrl + u/ctrl + k 从光标处向前（向后）删除指令符

ctrl + a/ctrl + e 让光标移动到整个指令符的最前面（最后面）

# shell 变量

mail指令会主动的取用MAIL这个变量。

不同用户的MAIL变量值是不同的，比如用户vbird，他取得变量值就是/var/spool/mail/vbird

变量内容若有空白字符可使用双引号“”或单引号''将变量内容结合起来

区别：

双引号内的特殊字符如$等，可以保有原本的特性。

单引号内的特殊字符则仅为一般字符（纯文本）

在一串指令的执行中，还需要借由其他额外的指令所提供的信息时，可以用反单引号。

如果该变量需要在其他子程序执行，则需要以export来使变量变成环境变量export 变量名

取消变量的方法为使用unset：unset 变量名称，例如取消myname的设置：unset myname

输入env，可以查看当前的环境变量与常见环境变量说明

# 指令搜索顺序

  1. 以相对/绝对路径执行指令，例如“/bin/ls”或“./ls”；
  2. 由alias找到该指令来执行
  3. 由bash内置的（builtin）指令来执行；
  4. 通过$PATH这个变量的顺序搜寻到第一个指令来执行。

# bash的进站与欢迎讯息

可以将linux登录提示字符串写在/etc/issue中，比如可以返回系统的版本信息，当前时间等。

issue内的各代码意义
\d 本地端时间的日期；
\l 显示第几个终端机接口；
\m 显示硬件的等级（i386/i486/i586/i686...）；
\n 显示主机的网络名称；
\O 显示domain name；
\r 操作系统的版本（相当于uname-r）;
\t 显示本地端时间的时间；
\S 操作系统的名称；
\v 操作系统的版本

例：

\S	（terminal:	\l）
Date:	\d	\t 
Kernel	\r	on	an	\m 
Welcome!

欢迎信息或通知讯息可以写在/etc/motd中，这样当使用者（包括所有的一般账号与root）登录主机后，就会显示讯息。
