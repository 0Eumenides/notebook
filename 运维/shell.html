# shell

# 介绍

Shell 是一种计算机操作系统的用户界面，它是一种命令行界面，可以通过输入命令来控制计算机。Shell 可以与操作系统进行交互，使用户能够执行各种操作，例如创建、删除和修改文件，以及管理进程等。本文将介绍 Shell 的基础知识，以帮助初学者快速入门。

# Shell 的类型

Shell 有许多不同的类型，其中最常用的是 Bash，它是 Linux 和 macOS 等操作系统默认的 Shell。其他常见的 Shell 类型包括 Zsh 和 Ksh 等。每种 Shell 都有自己的命令和语法，但大多数命令都是相同的。在本文中，我们将使用 Bash 作为示例。

# 脚本执行方式

Shell 脚本可以通过 bash 命令或source命令或 ./ 运算符来执行，./使用的更多。

命令特点
bash在新的 Shell 中执行脚本，不会影响当前 Shell 的环境变量和工作目录
source在当前 Shell 中执行脚本，会影响当前 Shell 的环境变量和工作目录
./在新的 Shell 中执行脚本，不会影响当前 Shell 的环境变量和工作目录

也就是说source会使用当前shell环境的变量，bash和./则不会，需要当前环境变量为export

Shell 中的 export 命令用于将值分配给变量，同时将变量标记为环境变量，这意味着它们在脚本中的所有命令都可用，并且在子 Shell 中也可用。没有使用 export 命令的变量仅在当前 Shell 会话中可用，并且在子 Shell 中不可用。

# Shell变量

# Shell 变量的命名要求

Shell 变量的命名遵循以下规则：

  • 变量名以字母或下划线开头，不能以数字开头。
  • 变量名中可以包含字母、数字和下划线。
  • 变量名必须是大小写敏感的。
  • 变量名不能是 Shell 的关键字。

赋值时等号两边不能有空格

# Shell 变量的类型

Shell 变量的类型包括：

  • 环境变量：使用 export 命令将值分配给变量，同时将变量标记为环境变量，这意味着它们在脚本中的所有命令都可用，并且在子 Shell 中也可用。
  • 局部变量：变量在 Shell 脚本中定义，但未使用 export 命令分配给它们，这意味着它们仅在当前 Shell 会话中可用，并且在子 Shell 中不可用。
  • Shell 变量：Shell 变量是由 Shell 已定义的变量，例如 Shell 的特殊变量 $0、$1 、$HOME、$PWD、$SHELL、$USER等。这些变量在 Shell 脚本中具有不同的作用和范围，可以根据需要选择使用。

Shell 变量的类型包括以下几种：

  • 字符串：默认类型，用于存储文本。可以使用单引号或双引号来定义字符串变量。使用双引号时，可以在字符串中使用变量，变量会在字符串中被替换为其值。
  • 整数：用于存储整数。整数变量可以使用 declare 或 typeset 命令定义，也可以使用赋值语句定义。
  • 数组：用于存储多个值。数组变量使用小括号来定义，其中每个值之间用空格分隔。可以使用下标来访问数组变量中的值，下标从 0 开始。

在 Shell 中，变量无需事先声明其类型。根据变量的值，Shell 会自动确定其类型。

Shell 中的数组变量可以存储多个值。数组变量的定义格式如下：

array_name=(value1 value2 ... valuen)

其中，array_name 是数组变量的名称，value1、value2 等是数组变量中的值，多个值之间用空格分隔。

访问数组变量的值可以使用下标，下标从 0 开始。例如，要访问数组变量 array_name 中的第一个值，可以使用以下语法：

${array_name[0]}

使用 ${#array_name[@]} 来输出数组变量的长度。例如，要输出数组变量 array_name 的长度，可以使用以下语法：

${#array_name[@]}

Shell 中有一些特殊变量，用于保存 Shell 的状态和其他信息。以下是一些常见的特殊变量：

  • $0：Shell 脚本的名称。
  • $1、$2、$3 等：Shell 脚本的参数。
  • $#：Shell 脚本的参数个数。
  • $*：所有参数列表。把所有参数看成一个整体
  • $@：所有参数列表，每个参数都用双引号括起来。分开对待
  • $?：上一个命令的退出状态。

# 变量的执行方式

变量可以通过 $ 符号和变量名来调用。例如，要调用变量 my_var，可以使用以下语法：

$my_var

在字符串中使用变量可以使用双引号来定义字符串变量，变量会在字符串中被替换为其值。例如，要在字符串中使用变量 my_var，可以使用以下语法：

"Hello, $my_var!"

如果不希望变量在字符串中被替换为其值，可以使用单引号来定义字符串变量。例如，要在字符串中使用变量 my_var，但不希望它在字符串中被替换为其值，可以使用以下语法：

'Hello, $my_var!'

可以使用 ${#variable} 来输出变量的长度。例如，要输出变量 my_var 的长度，可以使用以下语法：

${#my_var}

可以使用 ${variable:start:length} 来截取变量。其中，start 表示要截取的起始位置，length 表示要截取的长度。例如，要从变量 my_var 中截取前 5 个字符，可以使用以下语法：

${my_var:0:5}

要截取从第 5 个字符开始的后面的所有字符，可以使用以下语法：

${my_var:4}

其中，数字 4 表示从第 5 个字符开始。

# 只读变量

在 Shell 中，可以使用 readonly 命令将变量标记为只读。一旦变量被标记为只读，就不能再次分配给它。使用 readonly 命令的语法如下：

readonly variable_name

其中，variable_name 是要标记为只读的变量的名称。

标记为只读的变量在 Shell 脚本中非常有用，可以防止不小心更改变量的值。

# 注释

在 Shell 中，单行注释以 # 开头，多行注释可以使用以下语法：

:<EOF
这是一个多行注释。
这里可以写多行注释内容。
EOF

:<EOF 表示开始多行注释，EOF 表示结束多行注释。在 :<EOF 和 EOF 之间的所有内容都将被视为注释，不会被 Shell 解释或执行。这种语法非常有用，可以帮助开发人员在代码中添加注释和文档，提高代码的可读性和可维护性。

# Shell运算

在 Shell 中，运算符用于执行各种算术和逻辑运算。以下是一些常用的 Shell 运算符：

  • 算术运算符：用于执行算术运算，例如加、减、乘、除等。常见的算术运算符包括 +、、、/、% 等。
  • 关系运算符：用于比较两个值之间的关系，例如相等、大于、小于等。常见的关系运算符包括 =、!=、eq、ne、gt、lt、ge、le 等。
  • 布尔运算符：用于执行布尔运算，例如逻辑与、逻辑或、逻辑非等。常见的布尔运算符包括 &&、||、! 等。
  • 位运算符：用于执行位运算，例如按位与、按位或、按位异或等。常见的位运算符包括 &、|、^ 等。
  • 赋值运算符：用于将值分配给变量。常见的赋值运算符包括 =、+=、=、=、/=、%= 等。

除了这些常见的运算符之外，Shell 还有许多其他类型的运算符，例如字符串运算符、文件测试运算符等。

在 Shell 脚本中使用运算符时，需要注意运算符周围的空格。一般来说，运算符两侧应该留出空格，以使代码更易于阅读和理解。例如，以下是一个使用算术运算符的示例：

result=$(($a + $b))
result=`expr $a + $b`
result=$[a + b]

echo $result   # 输出 3

在这个示例中，$((1 + 2)) 使用算术运算符将 1 和 2 相加，然后将结果分配给变量 result。

注意

  • 乘号(*)前边必须加反斜杠()才能实现乘法运算；
  • expr 不支持浮点数运算。如果需要进行浮点数计算，可能需要使用其他工具，比如 bc。

而逻辑运算符需要使用[]来调用，如if [$a -ge $b]，也可以使用test :

test命令在Shell中用于检查文件的类型和比较值。它可以测试文件是否存在（-e），是否为普通文件（-f），是否为目录（-d），是否为空（-s），是否可读（-r），是否可写（-w），是否可执行（-x），以及比较运算符（-eq，-ne，-gt，-lt，-ge，-le）。可以使用[ ]或[[ ]]替代，但需要在中间留出空格。

# echo

echo 命令用于在 Shell 中打印文本。例如，要打印 "Hello, world!"，可以使用以下语法：

echo "Hello, world!"

echo 命令还可以使用转义字符来打印特殊字符，例如换行符和制表符。以下是一些常用的转义字符：

  • \\n：换行符
  • \\t：制表符
  • \\\\：反斜杠
  • \\"：双引号

例如，要打印一个包含制表符和换行符的字符串，可以使用以下语法：

echo "Hello,\\tworld!\\nHow are you today?"
echo `pwd`

# Shell 流程控制

在 Shell 中，可以使用各种控制结构来控制程序的流程。以下是一些常见的控制结构：

# if 语句

if 语句用于测试一个条件，并根据条件的结果执行不同的命令。以下是一个 if 语句的基本语法：

if [ condition1 ]
then
    command1
elif [ condition2 ]
then
		command2
else
    command3
fi

# for 循环

for 循环用于在 Shell 中迭代一个列表，并执行相应的命令。以下是一个 for 循环的基本语法：

for variable in item1 item2 ... itemN
do
    command1
    command2
    ...
done

其中，variable 是要迭代的变量，list 是要迭代的列表，command1、command2 等是要在每次迭代时执行的命令。

# while 循环

while 循环用于在 Shell 中重复执行命令，直到条件为假。以下是一个 while 循环的基本语法：

while condition
do
    command1
    command2
    ...
done

# until 循环

until 循环用于在 Shell 中重复执行命令，直到条件为真。以下是一个 until 循环的基本语法：

until condition
do
    command1
    command2
    ...
done

# case 语句

case 语句用于测试一个变量，并根据变量的值执行不同的命令。以下是一个 case 语句的基本语法：

case variable in
value1)
    command1
    command2
    ...
    ;;
value2)
    command3
    command4
    ...
    ;;
*)
    command5
    command6
    ...
    ;;
esac

其中，variable 是要测试的变量，value1、value2 等是变量可能的值，可以用read来读取变量，如read variable

在 Shell 中，可以使用 break 和 continue 命令来控制循环的执行。

# basename 和 dirname 函数

basename 函数用于获取指定路径中的文件名部分，dirname 函数用于获取指定路径中的目录部分。

# basename 函数

basename 函数的语法如下：

basename path [suffix]

其中，path 表示要获取文件名的路径，suffix 表示要去除的后缀。

如果不指定 suffix 参数，则 basename 函数会返回 path 中的文件名部分。

如果指定 suffix 参数，则 basename 函数会从 path 中去除指定的后缀，然后返回剩余的部分。

# dirname 函数

dirname 函数的语法如下：

dirname path

其中，path 表示要获取目录的路径。

dirname 函数会返回 path 中的目录部分。

如果 path 中不包含目录部分，则 dirname 函数会返回 .。

basename 和 dirname 函数在 Shell 脚本中非常有用，可以帮助开发人员处理文件和路径相关的任务，提高代码的可读性和可维护性。

# 自定义函数

在 Shell 中，可以使用函数来组织和重用代码。以下是一个函数的基本语法：

function_name () {
    command1
    command2
    ...
    return value
}

其中，function_name 是函数的名称，command1、command2 等是要在函数中执行的命令，value 是函数的返回值。

要调用函数，可以使用以下语法：

function_name arg1 arg2 ...

其中，arg1、arg2 等是函数的参数。调用时使用$1,$2,${10}来调用

函数还可以使用局部变量。局部变量只在函数内部可见，并在函数执行完毕后被删除。要定义局部变量，可以使用 local 命令。例如：

function my_func () {
    local my_var=123
    echo "my_var is $my_var"
}

my_func   # 输出 "my_var is 123"
echo "my_var is $my_var"   # 报错：my_var: 未找到命令

# 开机运行

  1. 创建一个需要运行的脚本，确保脚本具有可执行权限

    1. 创建一个新的 Systemd 服务文件。请使用文本编辑器创建一个新文件，例如 /etc/systemd/system/my_script.service，并添加以下内容：
    [Unit]
Description=My custom startup script
After=network.target

[Service]
ExecStart=/path/to/your/script.sh
Type=oneshot
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target

    请确保将 **ExecStart**的路径替换为您的实际脚本路径。

  3. 通知 Systemd 重新加载配置：

    sudo systemctl daemon-reload

  4. 启用新创建的服务，使其在开机时自动运行：

    sudo systemctl enable my_script.service

现在，每次系统启动时，/path/to/your/script.sh 都会自动运行。如果您需要手动启动、停止或查看服务状态，可以使用以下命令：

  • 启动服务：sudo systemctl start my_script.service
  • 停止服务：sudo systemctl stop my_script.service
  • 查看服务状态：sudo systemctl status my_script.service

# sed命令

sed 命令用于在 Shell 中对文本文件进行替换和编辑操作。以下是 sed 命令的基本语法：

sed [options] 'command' file

其中，options 是一些可选的命令行选项，command 是要执行的 sed 命令，file 是要处理的文件名。

sed 命令支持许多不同的命令，以下是一些常用的命令：

  • s/regexp/replacement/g：用 replacement 替换匹配到的 regexp。
  • d：删除匹配到的行。
  • p：打印匹配到的行。
  • i：在匹配到的行之前插入新行。
  • a：在匹配到的行之后添加新行。
  • r：读取文件并将其内容附加到匹配到的行。
  • w：将匹配到的行写入到另一个文件中。
  • y/source/dest/：将 source 中的字符替换为 dest 中的字符。

例如，以下 sed 命令将替换 file.txt 中的所有 foo 为 bar 并将结果打印到终端：

sed 's/foo/bar/g' file.txt

可以使用 sed 命令来添加新行

sed -i '/pattern/a new line' file.txt

其中，pattern 是要查找的模式，new line 是要添加的新行内容，file.txt 是要处理的文件名。该命令将在 file.txt 中查找包含 pattern 的行，并在其后面添加一行 new line。

使用 sed 命令来删除匹配到的行的语法如下：

sed '/pattern/d' file.txt
sed '5d' file.txt

其中，pattern 是要匹配的模式，file.txt 是要处理的文件名。该命令将在 file.txt 中查找包含 pattern 的行，并删除这些行。第二个命令是删除第五行

# Sort

sort 命令用于在 Linux 中对文本文件进行排序。以下是 sort 命令的基本语法：

sort [options] file

其中，options 是一些可选的命令行选项，file 是要排序的文件名。

sort 命令支持许多不同的选项，以下是一些常用的选项：

  • r：反向排序。
  • n：按数字顺序排序。
  • u：去重排序，仅输出不同的行。
  • k：按指定字段进行排序。
  • t：指定字段分隔符。

sort 命令非常有用，可以帮助开发人员处理文本文件相关的任务，提高代码的可读性和可维护性。

# wget

wget 命令用于在 Shell 中下载文件。以下是 wget 命令的基本语法：

wget [options] url

其中，options 是一些可选的命令行选项，url 是要下载的文件的 URL。

wget 命令支持许多不同的选项，以下是一些常用的选项：

  • O：指定下载文件的名称。
  • q：安静模式，不输出任何信息。
  • c：继续下载未完成的文件。
  • r：递归下载。
  • np：不下载父级链接。
  • nH：不创建主机名目录。

例如，以下命令将下载 URL 为 http://example.com/file.zip 的文件并将其保存为 myfile.zip：

wget -O myfile.zip <http://example.com/file.zip>

# 定时脚本

  1. 输入**crontab -e命令以编辑当前用户的cron**表。这将打开一个文本编辑器，您可以在其中添加或修改定时任务。

  2. 在编辑器中，添加一行来定义您的定时任务。这一行应包含五个时间字段，后跟要执行的命令或脚本。时间字段的顺序如下：

    • 分钟（0-59）
    • 小时（0-23）
    • 月份的第几天（1-31）
    • 月份（1-12）
    • 星期几（0-7，其中0和7都表示星期日）

    例如，如果您想要每天早上9点运行名为**my_script.sh**的脚本，您应该添加以下行（假设脚本位于您的主目录中）：

    0 9 * * * /bin/bash /path/to/your/script/my_script.sh

    如果要定时一次，如每两小时执行一次

    0 */2 * * * /bin/bash /path/to/your/script/my_script.sh

  3. 保存并关闭编辑器。**cron将自动加载更新后的cron**表，并按照您定义的时间间隔执行Shell脚本。

在某些Linux系统（如某些Docker容器和轻量级发行版）中，**cron和crontab命令可能未安装。在这种情况下，您需要手动安装cron包。以下是在Debian或Ubuntu系统上安装cron**的方法：

apt-get update
apt-get install cron

对于CentOS或RHEL系统：

yum install cronie
