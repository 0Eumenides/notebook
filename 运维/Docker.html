# Docker

特性普通虚拟机Docker
跨平台通常只能在桌面级系统运行，例如 Windows/Mac，无法在不带图形界面的服务器上运行支持的系统非常多，各类 windows 和 Linux 都支持
性能性能损耗大，内存占用高，因为是把整个完整系统都虚拟出来了性能好，只虚拟软件所需运行环境，最大化减少没用的配置
自动化需要手动安装所有东西一个命令就可以自动部署好所需环境
稳定性稳定性不高，不同系统差异大稳定性好，不同系统都一样部署方式

# 常用命令

# docker version

查看版本信息

用法: docker version

# docker info

查看docker的系统信息,包括镜像和容器的数量

用法: docker info

# docker search

搜索镜像

示例：docker search mysql

# docker run

运行一个新的容器

用法: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

示例: docker run -it ubuntu /bin/bash

# docker start

启动一个或多个已经被停止的容器

用法: docker start [OPTIONS] CONTAINER [CONTAINER...]

示例: docker start my_container

# docker stop

停止一个或多个正在运行的容器

用法: docker stop [OPTIONS] CONTAINER [CONTAINER...]

示例: docker stop my_container

# docker restart

重启一个或多个容器

用法: docker restart [OPTIONS] CONTAINER [CONTAINER...]

示例: docker restart my_container

# docker ps

列出所有正在运行的容器

用法: docker ps [OPTIONS]

示例: docker ps -a

# docker images

列出所有本地镜像

用法: docker images [OPTIONS] [REPOSITORY[:TAG]]

示例: docker images

# docker rm

删除一个或多个容器

用法: docker rm [OPTIONS] CONTAINER [CONTAINER...]

示例: docker rm my_container

# docker rmi

删除一个或多个镜像

用法: docker rmi [OPTIONS] IMAGE [IMAGE...]

示例: docker rmi my_image

# 镜像命令

# docker build

通过 Dockerfile 构建镜像

用法：docker build [OPTIONS] PATH | URL | -

示例：docker build -t my_image:latest .

# docker push

将镜像推送到 Docker Hub

用法：docker push [OPTIONS] NAME[:TAG]

示例：docker push my_image:latest

# docker pull

从 Docker Hub 拉取镜像

用法：docker pull [OPTIONS] NAME[:TAG|@DIGEST]

示例：docker pull my_image:latest

# docker tag

给镜像打标签

用法：docker tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]

示例：docker tag my_image:latest my_user/my_image:latest

# docker commit

用于将容器保存为新的镜像

用法：docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

示例：docker commit my_container my_image:1.0

# 容器命令

# docker exec

在正在运行的容器中执行命令

用法：docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

示例：docker exec -it my_container /bin/bash

# docker logs

查看容器日志

用法：docker logs [OPTIONS] CONTAINER

示例：docker logs my_container

# docker inspect

查看容器详细信息

用法：docker inspect [OPTIONS] NAME|ID [NAME|ID...]

示例：docker inspect my_container

# docker attach

进入正在运行的容器

用法：docker attach [OPTIONS] CONTAINER

示例：docker attach my_container

# docker cp

在容器和主机之间复制文件

用法：docker cp [OPTIONS] CONTAINER:SRC_PATH DEST_PATH|- docker cp [OPTIONS] SRC_PATH|- CONTAINER:DEST_PATH

示例：docker cp my_container:/root/test.txt /tmp

# Dockerfile

Dockerfile 是一个文本文件，用于定义构建 Docker 镜像的步骤和规则。Dockerfile 中包含了一系列指令，Docker 使用这些指令按顺序执行，从基础镜像开始，逐步添加修改，最终生成一个新的镜像。以下是一些常用的 Dockerfile 指令及其用途：

  1. FROM：指定基础镜像。这是 Dockerfile 的第一条指令，指明从哪个镜像开始构建。例如：FROM ubuntu:20.04。
  2. MAINTAINER：指定镜像的维护者信息。虽然这个指令已被弃用，但仍然可以使用。推荐使用 LABEL 代替。例如：MAINTAINER John Doe <john.doe@example.com>。
  3. LABEL：为镜像添加元数据，可以设置多个键值对。例如：LABEL maintainer="John Doe <john.doe@example.com>" version="1.0"。
  4. RUN：执行命令，并将结果提交为新的镜像层。常用于安装软件包、创建目录等操作。例如：RUN apt-get update && apt-get install -y nginx。
  5. CMD：指定容器启动时默认执行的命令。一个 Dockerfile 中只能有一个 CMD 指令。如果指定了多个，只有最后一个会生效。如果在运行容器时提供了命令，则会覆盖 CMD 指令。例如：CMD ["nginx", "-g", "daemon off;"]。
  6. ENTRYPOINT：配置容器启动时执行的命令。与 CMD 类似，但 ENTRYPOINT 不会被 docker run 提供的命令覆盖。可以与 CMD 结合使用，CMD 中的命令会作为 ENTRYPOINT 的参数。例如：ENTRYPOINT ["java", "-jar"] 和 CMD ["my-app.jar"]。
  7. WORKDIR：设置后续 RUN、CMD、ENTRYPOINT、COPY 和 ADD 指令的工作目录。例如：WORKDIR /app。
  8. COPY：将宿主机上的文件或目录复制到镜像中。例如：COPY ./app /app。
  9. ADD：与 COPY 类似，但可以解压 TAR 文件，或者从 URL 下载文件。例如：ADD https://example.com/myfile.tar.gz /app。
  10. EXPOSE：声明容器运行时监听的端口。注意，这不会自动映射端口到宿主机，需要使用 docker run -p 参数映射。例如：EXPOSE 80。
  11. VOLUME：在容器中创建挂载点。这允许将宿主机上的目录或文件挂载到容器中。例如：VOLUME /data。
  12. USER：指定后续 RUN、CMD 和 ENTRYPOINT 指令使用的用户。例如：USER myuser。
  13. ARG：定义构建参数。这些参数在构建过程中可以使用，也可以在运行时通过 -build-arg 参数设置。例如：ARG DEBIAN_FRONTEND=noninteractive。
  14. ENV：设置环境变量。这些变量将在容器运行时生效，也可以在构建过程中使用。例如：ENV JAVA_HOME=/usr/lib/jvm/java-11-openjdk。
  15. SHELL：指定后续 RUN 指令使用的默认 shell。例如：SHELL ["/bin/bash", "-c"]。
  16. ONBUILD：为基础镜像设置触发器。当有其他镜像以当前镜像为基础镜像时，ONBUILD 指令将被执行。例如：ONBUILD RUN npm install。
  17. HEALTHCHECK：用于检查容器内服务的健康状况。可以指定一个命令，定期执行以检查服务是否正常运行。例如：HEALTHCHECK --interval=5m --timeout=3s CMD curl -f http://localhost:80/ || exit 1。
  18. STOPSIGNAL：设置停止容器时发送给容器内主进程的信号。例如：STOPSIGNAL SIGTERM。
  • 使用 RUN 在构建过程中执行命令，例如安装依赖、创建文件等。
  • 使用 CMD 设置容器默认行为，当用户没有提供特定命令时执行。
  • 使用 ENTRYPOINT 将镜像作为可执行程序，接受用户提供的参数。可以与 CMD 结合使用，提供默认参数。

要编写 Dockerfile，首先确定基础镜像，然后根据需求添加各种指令。一个典型的 Dockerfile 可能包括：

  • 选择基础镜像（FROM）
  • 设置工作目录（WORKDIR）
  • 复制或添加文件（COPY 或 ADD）
  • 运行命令，例如安装依赖（RUN）
  • 暴露端口（EXPOSE）
  • 设置环境变量（ENV）
  • 指定启动命令（CMD 或 ENTRYPOINT）

以下是一个简单的示例：

要编写一个 Dockerfile 为 Python 项目安装依赖，可以参考以下步骤：

  1. 选择一个基础镜像。通常，可以使用官方的 Python 镜像，例如 python:3.8 或 python:3.9。
  2. 设置工作目录，以便在容器中创建一个目录来存放项目文件。
  3. 复制 requirements.txt 文件到容器中的工作目录。
  4. 使用 RUN 指令安装 requirements.txt 中的依赖。
  5. 复制项目的其他文件到工作目录。
  6. 暴露需要的端口（如果有）。
  7. 设置项目的启动命令。
# 基础镜像
FROM python:3.9

# 设置工作目录
WORKDIR /app

# 复制 requirements.txt
COPY requirements.txt .

# 安装依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制项目文件
COPY . .

# 暴露端口（如果有）
EXPOSE 8000

# 启动命令
CMD ["python", "app.py"]

根据你的项目情况进行调整，例如使用不同的 Python 版本、暴露其他端口或者使用不同的启动命令。确保在 Dockerfile 所在目录运行 docker build 命令时，requirements.txt 和其他项目文件也在该目录中。

# 目录挂载

  • Docker运行后，本地修改了项目代码需要同步到docker容器中
  • 容器中产生的log文件、数据库备份文件需要在本地同步保存，避免容器删除后导致文件的丢失

几种挂载方式：

  • bind mount 可以将宿主机目录直接映射到容器内，适合挂载代码目录和配置文件。例如：docker run -v /host/path:/container/path image_name，其中 /host/path 是宿主机目录，/container/path 是容器内目录。
  • volume 由 Docker 创建和管理，创建在宿主机上，因此删除容器时不会丢失数据。它们更高效，适用于存储数据库数据。例如：docker run -v volume_name:/container/path image_name，其中 volume_name 是卷的名称，/container/path 是容器内目录。
  • tmpfs mount 适合存储临时文件，将数据存储在宿主机内存中。不适合多个容器共享。例如：docker run --mount type=tmpfs,destination=/container/path image_name，其中 /container/path 是容器内目录。

# 多容器通信

创建一个自定义网络

docker network create my-network

接下来，运行第一个容器（例如，一个名为**container-1的Redis容器）并将其连接到my-network**网络：

docker run -d --name container-1 --net my-network redis

然后，运行第二个容器（例如，一个名为**container-2的Python容器），并将其连接到my-network**网络：

docker run -it --name container-2 --net my-network python bash

现在，这两个容器都连接到了同一个网络**my-network。在container-2内部，可以使用container-1的容器名作为其主机名来访问container-1**提供的服务。

import redis

# 使用container-1的名称作为主机名
# 如果容器连接到网络时设置了--network-alias参数，则使用这个别名进行访问
client = redis.Redis(host='container-1', port=6379)

client.set('key', 'value')
print(client.get('key'))

# Docker-Compose

当项目需要多个依赖时，需要管理的容器就更加多，可以使用 docker-compose 把项目的多个服务集合到一起，一键运行，统一管理。

要把项目依赖的多个服务集合到一起，需要编写一个docker-compose.yml文件，描述依赖哪些服务。

version: "3.7"  # 指定docker-compose文件的版本为3.7

services:  # 定义服务部分
  app:  # 定义名为app的服务
    build: ./  # 从当前目录（包含Dockerfile）构建app服务的镜像
    ports:  # 定义端口映射
      - 80:8080  # 将容器内的8080端口映射到宿主机的80端口
    volumes:  # 定义数据卷
      - ./:/app  # 将当前目录（宿主机）挂载到容器的/app目录
    environment:  # 定义环境变量
      - TZ=Asia/Shanghai  # 设置时区为亚洲/上海
  redis:  # 定义名为redis的服务
    image: redis:5.0.13  # 指定使用redis:5.0.13镜像
    volumes:  # 定义数据卷
      - redis:/data  # 将名为redis的数据卷挂载到容器的/data目录
    environment:  # 定义环境变量
      - TZ=Asia/Shanghai  # 设置时区为亚洲/上海

volumes:  # 定义数据卷部分
  redis:  # 定义名为redis的数据卷

在docker-compose.yml 文件所在目录，执行：docker-compose up就可以跑起来了。加上-d参数可以在后台运行

要停止并删除所有服务，使用：docker-compose down

# 数据备份与迁移

如果是用bind mount直接把宿主机的目录挂进去容器，那迁移数据很方便，直接复制目录就好了

如果是用volume方式挂载的，由于数据是由容器创建和管理的，需要用特殊的方式把数据弄出来。

备份：

  • 运行一个 ubuntu 的容器，挂载需要备份的 volume 到容器，并且挂载宿主机目录到容器里的备份目录。
  • 运行 tar 命令把数据压缩为一个文件
  • 把备份文件复制到需要导入的机器

导入：

  • 运行 ubuntu 容器，挂载容器的 volume，并且挂载宿主机备份文件所在目录到容器里
  • 运行 tar 命令解压备份文件到指定目录
